移动端布局

2018-07-20 web前端 HTML5+CSS3 评论 字数统计: 492(字) 阅读时长: 1(分)

1.viewport视口

1. 可视视口：固定大小的，跟手机设备一样。在上面。

2. 布局视口：可调节大小的，默认980，压缩到可视视口的大小，在下面需要让布局视口改成跟可视视口一样大

  • width : device-width 动态的 ，设备414 -> 414 设备375 -> 375。width : 414 可以给固定大小，99%网站不会这么设置，因为不好适配。
  • height : 不去设置。
  • initial-scale : 1.0 初始缩放
  • minimum-scale : 1.0 最小拉伸
  • maximum-scale : 1.0 最大拉伸
  • user-scalable : no 不允许用户拉伸

2.移动端布局方案

流式布局

  • 主要是当屏幕缩小时，里面的元素之间距离减少，主要使用弹性布局

    • 好处： 大屏幕下显示更多的内容。
    • 坏处：宽屏下比例会有一些不协调。

rem布局

  • 当屏幕缩小时，内部元素也在减小，多使用rem单位，通过px转rem来实现

    • 好处： 不同设备下看起来比较协调。
    • 坏处： 显示的内容并不会在大屏幕展示的内容多。

  • em : 是一个相对单位，1em等于当前元素或父元素的font-size值

  • rem : 是一个相对单位，1rem等于根元素的font-size值。1rem == 1个根元素的font-size大小 就是 1个html标签的font-size大小

  • 如何做到html的font-size动态化

      1. 通过JS （JavaScript）

      1. 通过vw （vw是尺寸单位）

        • 相对单位：表示把屏幕自动分成了100vw宽和100vh高。vw / vh : 把屏幕分为100份，1vw等于屏幕宽的1%
        • 当设计师给我们的psd如果是其他尺寸的可以将其改为375尺寸，方便操作
        • 步骤：将尺寸改为375px,然后在页面头部加入html{font-size:26.66667vw}body{font-size:16px},然后就可以根据psd量取间距，最后通过工具将其转换为rem单位
